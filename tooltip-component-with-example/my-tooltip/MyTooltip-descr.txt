Конструктор создает один тултип, для создания нескольких нужно для каждого отдельно вызвать конструктор.
Инициализация:
let tooltip = new MyTooltip(trigger, userOptions);
    trigger - это селектор или элемент
options:
-- makeContent(triggerBtn) => {} - функция, принимает в качестве аргумента кнопку показывающую тултип. возвращает содержимое в виде элемента или строки
-- content - содержимое тултипа в виде строки или элемента;
-- transition - время анимации в ms, с такой же задержкой удаляется тултип из дом;
-- easing - название easing-функции;
--toolitpId - уникальный идентификатор тултипа(для настроек доступности);
--enableArrow - boolean, def=true;
--triggerEvent = string, def= 'click' можно через запятую указать несколько. Например 'click focus' 'focus hover'. Клик и ховер вместе не работают.
--onShow()=> {} - функция, действия при показывании тултипа;
--onHide()=> {} -функция, действия при сокрытии тултипа;
--tooltipClassName: string, def='tooltip'
--arrowClassName: string, def = 'tooltip-arrow';
--tooltipVisibleClassName: string, def = 'tooltip--visible';


Notes:
! В качестве id не указывать "0", т.к. при проверке 0 = false и id не устанавливается.
!! Классы можно заменить, но, чтобы это не повлияло на работу нужно перенести дефолтные стили в новые классы

Проблемы:
- если triggerEvent: 'hover', при быстром движении мыши могут создаваться пустые дубликаты.

Требует доработки:
